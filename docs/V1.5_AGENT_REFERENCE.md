## What You Need to Know About V1.5

### The Big Picture
V1.5 transforms the capacity modeler from **linear-only** (V1) to **complex production networks** (V1.5).

**V1.5 enables:**
- Assemblies (merges): 4 units Part A + 1 unit Part B → 1 Assembly
- Quality/waste: 95% good output, 5% scrap (handled via yield)
- Byproducts & secondary markets: 80% to Customer A, 20% to Secondary Market
- Complex chains: Assembly → Testing → Split (95% pass, 5% fail)

**V1.5 does NOT include:**
- Authentication (next release)
- Import/Export (next release)
- Operational calendars or shift templates
- Multi-product optimization
- Stochastic simulation or rework loops

## Critical Technical Decisions

### 1. Graph Topology
- **One Sink node** (single final output)
- **Fork-join allowed** (node with multiple inputs AND outputs, e.g., Assembly → Test)
- **DAGs only** (no cycles)
- **Scrap edges are dead-ends** (excluded from demand propagation)

### 2. Split Semantics
Two types of output edges:
- **Real output:** Carries downstream demand (e.g., "95% good product" or "80% Customer A")
- **Loss/scrap:** Visual only, no demand signal (e.g., "5% defective waste")

Both sum to 100% and are handled via yield + split ratios.

### 3. Demand Propagation
```
1. Topological sort on REAL edges only (scrap edges ignored)
2. Propagate backward from Sink to Sources
3. At merges: multiply required throughput by BOM ratio
4. At splits with multiple real outputs: demand sums across all paths
5. Apply yield at every node: requiredInput = requiredOutput / (yield/100)
```

### 4. Material Field
- Optional `outputMaterial?: string` for UI labeling only
- NO material compatibility validation in V1.5
- Examples: "Raw Steel", "Cut Steel", "Assembled Unit"

## Common Mistakes to Avoid

### ❌ Don't:
- Treat scrap edges as real outputs with demand
- Design split propagation assuming loss-only (we now support distribution)
- Allow fork-join in validation (it's now explicitly allowed)
- Add auth, import/export, or calendars (out of scope)
- Create rework loops or cycles (DAGs only)

### ✅ Do:
- Exclude scrap edges from topological sort
- Test fork-join scenarios explicitly (Assembly → Testing)
- Test multi-level cascading (Merge → Split → Merge)
- Validate split ratios sum to 100%
- Apply yield at every node during propagation

## Quick Troubleshooting

### "How do split ratios work?"
Split ratios define how a node's output is distributed:
- **[95% real, 5% scrap]** = 95% yield (good output); 5% waste (scrap, no demand)
- **[80% Customer A, 20% Secondary]** = distribution to two real markets (both carry demand)

### "Can a node both merge and split?"
**Yes!** Fork-join is allowed. Example: Assembly (3 inputs) → Testing (95% pass, 5% fail).

### "What about rework loops?"
**Not allowed in V1.5.** No cycles in the graph. Rework is deferred to a later release.

### "Where does material identity matter?"
Only in the UI for labeling (Source outputs "Raw Steel", Process outputs "Cut Steel"). No validation enforced.

### "What if BOM ratios are missing?"
**Validation error.** All incoming edges on a merge node MUST have BOM ratios defined.
